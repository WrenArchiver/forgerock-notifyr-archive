ba27e656e73 DOCS-436 Support element exclusions based on the document name

This patch makes it possible to exclude an element from a document.
To use this feature, set a `condition` attribute to `-doc-name`,
where `doc-name` is the name of the document.

The following example excludes a `<para>` from the OEM Release Notes:

```
<para condition="-oem-release-notes">Not for OEM Release Notes</para>
```

This is useful for shared content, where some of the shared content
should not be present in all documents that XInclude the shared content file.

This patch also adds a property `isXmlInlined`, default: `false`,
defining whether to inline XML during preprocessing.

If your document XIncludes XML files from other folders,
you must apply XML inlining in the plugin configuration to use this feature.
To do so, set `<isXmlInlined>true</isXmlInlined>` or `-DisXmlInlined=true`.
Inlining is a preprocessing step.
Inlining replaces all XIncluded elements by the content of the files.
Inlining then removes all XML documents except the top-level DocBook source documents
(by default, all XML files not named `index.xml`).

This affects only the XML files. Images and other files are not moved.		 Mark 28 Sep - 08:25 AM
