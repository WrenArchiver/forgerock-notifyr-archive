1 + # ForgeRock Doc Maven Plugin
2 +
3 + Most core documentation sources for the open-source projects
4 + led by ForgeRock are DocBook XML and Asciidoc text files.
5 +
6 + The open-source projects led by ForgeRock use Apache Maven
7 + for builds and to maintain project dependencies.
8 +
9 + The `forgerock-doc-maven-plugin` and its dependencies
10 + make it possible to process core documentation sources
11 + and to build draft docs with Maven.
12 +
13 + The plugin started out as a wrapper for the
14 + [docbkx-tools](https://github.com/mimil/docbkx-tools) plugin,
15 + which drives the DocBook XSL stylesheets in Maven builds.
16 + It has since grown to perform a variety of pre-processing tasks.
17 +
18 + ## Get Started
19 +
20 + The `forgerock-doc-maven-plugin` is available
21 + from ForgeRock's Maven repository:
22 +
23 + ```
24 +   <properties>
25 +     <mavenRepoSnapshots>https://maven.forgerock.org/repo/snapshots</mavenRepoSnapshots>
26 +     <mavenRepoReleases>https://maven.forgerock.org/repo/releases</mavenRepoReleases>
27 +     <doc-tools.version>some.version.number</doc-tools.version>
28 +   </properties>
29 +   
30 +   <repositories>
31 +     <repository>
32 +       <id>forgerock-staging-repository</id>
33 +       <name>ForgeRock Release Repository</name>
34 +       <url>https://maven.forgerock.org/repo/releases</url>
35 +       <snapshots>
36 +         <enabled>false</enabled>
37 +       </snapshots>
38 +     </repository>
39 +     <repository>
40 +       <id>forgerock-snapshots-repository</id>
41 +       <name>ForgeRock Snapshot Repository</name>
42 +       <url>https://maven.forgerock.org/repo/snapshots</url>
43 +       <releases>
44 +         <enabled>false</enabled>
45 +       </releases>
46 +     </repository>
47 +   </repositories>
48 +
49 +   <build>
50 +     <plugins>
51 +       <plugin>
52 +         <groupId>org.forgerock.commons</groupId>
53 +         <artifactId>forgerock-doc-maven-plugin</artifactId>
54 +         <version>${doc-tools.version}</version>
55 +       </plugin>
56 +     </plugins>
57 +   </build>
58 +
59 + ```
60 +
61 + ## Build Your Own
62 +
63 + Build and run basic tests:
64 +
65 + ```
66 + mvn -Pprecommit install
67 + ```
68 +
69 + Also run integration tests:
70 +
71 + ```
72 + mvn -Pprecommit,it install
73 + ```
