1 + # ForgeRock Documentation Tools 3.2.3 Release Notes
2 +
3 + ForgeRock Documentation Tools is a catch all for the doc build artifacts,
4 + sites where we post release documentation,
5 + and the documentation about documentation.
6 +
7 + The link to the online issue tracker is
8 + <https://bugster.forgerock.org/jira/browse/DOCS>.
9 +
10 + These release notes are for the 3.2.3 maintenance release.
11 +
12 +
13 + ## Compatibility
14 +
15 + This release depends on artifacts in ForgeRock-internal repositories.
16 + The doc tools are therefore no longer useful outside the ForgeRock network.
17 +
18 + Some references to image locations in shared files and nested includes have changed
19 + for compatibility with BackStage. For details, see <https://bugster.forgerock.org/jira/browse/DOCS-422>.
20 +
21 + In a future release, PDF output may no longer be generated by default.
22 +
23 +
24 + ## Limitations
25 +
26 + Documentation builds depend on system support for UTF-8.
27 + The locale for the user running the documentation build must support UTF-8, too.
28 + For example:
29 +
30 +     LANG="en_US.UTF-8"
31 +
32 + Some systems set the locale to use ASCII by default.
33 + On such systems, it is *not sufficient* to set the JVM to use UTF-8,
34 + as in `-Dfile.encoding=UTF-8 -Dsun.jnu.encoding=UTF-8`.
35 + Lack of support for UTF-8 results in some UTF-8 characters being rendered
36 + as `???` or `??`.
37 +
38 + Jenkins hosts require a host setting under *Prepare an environment for the run*,
39 + Properties Content: `LC_ALL=en_US.UTF-8`.
40 +
