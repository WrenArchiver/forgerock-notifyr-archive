1 + # ForgeRock Documentation Tools 3.2.2 Release Notes
2 +
3 + ForgeRock Documentation Tools is a catch all for the doc build artifacts,
4 + sites where we post release documentation,
5 + and the documentation about documentation.
6 +
7 + The link to the online issue tracker is
8 + <https://bugster.forgerock.org/jira/browse/DOCS>.
9 +
10 + These release notes are for maintenance releases 3.2.1 and 3.2.2.
11 +
12 + ## Compatibility
13 +
14 + With the fix for DOCS-423, the `docbook-linktester` report file name
15 + has changed from `linktester.err` to `linktester.txt`.
16 + This should make it easier to see the content quickly in mail clients.
17 +
18 + ## Limitations
19 +
20 + Documentation builds depend on system support for UTF-8.
21 + The locale for the user running the documentation build must support UTF-8, too.
22 + For example:
23 +
24 +     LANG="en_US.UTF-8"
25 +
26 + Some systems set the locale to use ASCII by default.
27 + On such systems, it is *not sufficient* to set the JVM to use UTF-8,
28 + as in `-Dfile.encoding=UTF-8 -Dsun.jnu.encoding=UTF-8`.
29 + Lack of support for UTF-8 results in some UTF-8 characters being rendered
30 + as `???` or `??`.
31 +
32 +
33 + ## Improvements & New Features
34 +
35 + **DOCS-417: Use backslash as the escapeString for Maven filtering**
36 +
37 + To avoid Maven filtering resolving a reference to a defined property,
38 + write `\${version}` in the source of the document.
39 +
40 + **DOCS-413: Create a HTML cover page to point directly to built docs for each PR**
41 +
42 + The `site` build creates an unstyled `site/doc/index.html` page with links to Bootstrap HTML.
43 +
44 + **DOCS-394: Trim verbatim elements that contain inline markup**
45 +
46 + The previous, broken patch was reverted.
