92a8a240185 DOCS-390 Allow normal links to work in PDF

The fix for DOCS-379 caused a regression:
it prevented any `<link>` elements from being rendered in PDF.
This was due to overly aggressive template matching.

This patch matches only `<link>` elements
where the `xlink:href` value contains `../resources`.

Tested with `forgerock-doc-maven-plugin-test`.		 01 Mar 2016
