d6135dd6d44 CHF-95 Add the originalUri into UriRouterContext

We need that special URI because we had to generate redirects at some points in
the chain, when the request.uri has already been changed by some previous filters.
As the redirect has an absolute Location header that need to be resolvable by
the client/browser, we can't relay on request.uri and must keep a pointer on the
original request URI, as received by the servlet.		 14 Oct 2015
