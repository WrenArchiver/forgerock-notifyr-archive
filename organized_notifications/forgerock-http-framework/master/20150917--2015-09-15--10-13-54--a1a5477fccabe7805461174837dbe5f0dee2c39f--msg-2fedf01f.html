a1a5477fcca CHF-72 Json parser accept any valid JSON and fail with invalid content

Jackson already produces a `LinkedHashMap`, so there is no need to deal
with `JsonParser`/`JsonToken` in order to control the JSON Object mapped
Java type.

As a consequence, `readJson()` is now throwing exceptions for all invalid
content (including empty strings and other variations that were returning
`null` before).

`readJson()` is now stricter than before in regards to supported JSON content.		 15 Sep 2015
