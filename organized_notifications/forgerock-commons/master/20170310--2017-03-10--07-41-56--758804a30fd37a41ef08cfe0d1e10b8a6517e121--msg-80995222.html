8 + # Common Self Service
9 +
10 + The purpose of the commons self service (CSS) project is to provide a common approach to forgotten password and user registration.
11 +
12 + More information can be found here: https://wikis.forgerock.org/confluence/display/COMPLAN/CSS+-+Commons+Self+Service
13 +
14 + # Deploying the demo application
15 +
16 + First make sure it is built:
17 +
18 + ```
19 +     mvn clean install
20 + ```
21 +
22 + Then to get it running:
23 +
24 + ```
25 +     mvn jetty:run -f forgerock-selfservice-example/pom.xml -Dmailserver.username=? -Dmailserver.password=?
26 + ```
27 +
28 + It is necessary to give values to the above properties. These properties represent:
29 +
30 + ```
31 +     mailserver.username - mail server username (use your forgerock email address)
32 +     mailserver.password - mail server password (use your forgerock email password)
33 + ```
34 +
35 + Once the service is running you can visit this page:
36 +
37 + ```
38 +     http://localhost:9999/example
39 + ```
40 +
41 + First it is necessary to go through the self registration process to create some users. Once a user is created the password reset can be played with.
42 +
43 + To query all demo users run:
44 +
45 + ```
46 +     curl --request GET \
47 +       --url 'http://localhost:9999/example/selfservice/internal/users?_queryFilter=true' \

49 + ```
