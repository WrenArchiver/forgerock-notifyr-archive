1 + # Audit Benchmarks
2 +
3 + ### Description
4 +
5 + The `forgerock-audit-benchmark` module runs [JMH](http://openjdk.java.net/projects/code-tools/jmh/)
6 + benchmarks that are bootstrapped from unit test classes. During builds, test do **not** run by default,
7 + because their execution time may be lengthy and temporary files generated may be large. The results
8 + from each test class are printed to the command-line and written to a JSON file, which could be
9 + processed to detect performance improvements/regressions.
10 +
11 + Not that the JVM's temporary directory is used to store all generated files, and that these
12 + files/directories are deleted when a test completes. Cleanup may fail if an error occurs during
13 + execution of the tests.
14 +
15 + ### Running the Tests
16 +
17 + To run all tests, with default settings,
18 +
19 + ```
20 + cd ./audit/forgerock-audit-benchmark
21 +
22 + mvn clean test -DskipTests=false
23 + ```
24 +
25 + Run a single test with,
26 +
27 + ```
28 + mvn clean test -DskipTests=false -Dtest=CsvAuditEventHandlerWriteBenchmarkTest
29 + ```
30 +
31 + ### Command-Line Options
32 +
33 + The following command-line options are passed to the JMH runtime, which instantiates its own JVM
34 + instances.
35 +
36 + Option                   | Description
37 + ------------------------ | -------------------
38 + `-DwarmupIterations=10`  | Number of warmup iterations to run per benchmark (default 10)
39 + `-DmeasureIterations=10` | Number of measurement iterations to run per benchmark (default 10)
40 + `-Dforks=2`              | Number of forks to run per benchmark (default 2)
41 + `-Dthreads=4`            | Number of concurrent threads to use per benchmark (default 4)
42 + `-DperfReportDir=/myDir` | Directory to output JMH report named `[unitTestClassName].json`. The default output directory is defined in the Maven POM as `${project.build.directory}/reports/performance/`.
