1 + # Commons Backstage-Connect
2 +
3 + This project contains clients for communicating with Backstage-Connect services (e.g., Usage Service).
4 +
5 + ## JSON Web Token Structure
6 +
7 + The JSON Web Token contains the following data entries,
8 +
9 + - `v` header (string), which is the version of our JWT structure (e.g., `1.0`)
10 + - `key` claim (string), containing a unique customer-assigned identifier
11 + - `iat` claim (long integer), which is the standard JWT issued-at-time
12 + - [optional] `url` claim (string), which will override the service-URL, and is useful for testing
13 +
14 + ## JSON Web Token JAR
15 +
16 + To generate the JAR file, containing a JWT with the customer-assigned "key", use the following,
17 +
18 + ```
19 + jar cmf MANIFEST.MF forgerock-backstage-connect-config.jar backstage-connect.jwt
20 + ```
21 +
22 + The file `backstage-connect.jwt` is plain-text, containing a signed and encoded JSON Web Token (JWT). The `MANIFEST.MF`
23 + must contain OSGi metadata (e.g., `Bundle-SymbolicName`), in order to be read within an OSGi environment
24 + (e.g., OpenIDM).
25 +
26 + For example,
27 +
28 + ```
29 + Manifest-Version: 1.0
30 + Bundle-Description: ForgeRock Backstage Connect Client Config
31 + Bundle-License: Commercial
32 + Bundle-ManifestVersion: 2
33 + Bundle-Name: ForgeRock Backstage Connect Client Config
34 + Bundle-SymbolicName: org.forgerock.backstage.connect.config
35 + Bundle-Vendor: ForgeRock AS
36 + Bundle-Version: 1.0.0
37 + Implementation-Build: 0
38 + SCM-Revision: 0
39 + ```
40 +
41 + Note that this same JWT can alternatively be provided to products via the `BACKSTAGE_CONNECT_JWT` environment variable,
42 + for use cases such as Docker images. However, we consider the embedded JAR solution to be more customer friendly, for
43 + downloads directed from the Backstage Connect website.
