1 + # ForgeRock Common HTTP Framework (CHF)
2 +
3 + ## What is CHF?
4 +
5 + CHF (pronounced `chuff`), is the common HTTP framework used throughout ForgeRock products to expose and consume HTTP
6 + APIs in a container/client-agnostic way. This allows other ForgeRock Commons projects to build on top of an
7 + HTTP Framework without having to restrict the products using it to particular architectures - for servers, bindings for
8 + both [Servlet](https://java.net/projects/servlet-spec/) and [Grizzly](https://grizzly.java.net/) are
9 + provided, and for the client, both synchronous and asynchronous client bindings are provided.
10 +
11 + The main module for CHF is `chf-http-core`, which contains all the APIs for exposing request handlers, describing them,
12 + consuming them and filtering them. For CHF server applications, one of `chf-http-servlet` or `chf-http-grizzly` are then
13 + bound to the container. For clients, either `chf-client-apache-async` or `chf-client-apache-sync` are added to classpath
14 + depending on the type of operation required. In the case of either client or server, the bindings are not needed to be
15 + dependencies of the modules that provide or consume services - they just have to be included in the final applciation
16 + (and wired in appropriately for servers).
17 +
18 + ## Getting started
19 +
20 + You can quickly try out a [simple CREST HTTP application](json-resource-examples/src/main/java/org/forgerock/json/resource/http/examples)
21 + using the following commands in a terminal:
22 +
23 + ```
24 + $ git clone https://stash.forgerock.org/scm/commons/forgerock-commons.git
25 + $ cd forgerock-commons
26 + $ mvn clean:install
27 + $ cd http-framework/http-examples/http-servlet-example
28 + $ mvn jetty:run
29 + ```

31 + ## Implementing an application
32 +
33 + To create a CHF application, you must expose an `HttpApplication` which returns a `Handler` that will be used as the
34 + root of the application. A simple example is in the examples:
35 + [ExampleHttpApplication](http-examples/http-servlet-example/src/main/java/org/forgerock/http/servlet/example/ExampleHttpApplication.java)
36 +
37 + ```
38 + public class ExampleHttpApplication implements HttpApplication {
39 +
40 +     @Override
41 +     public Handler start() {
42 +         return new Handler() {
43 +             @Override
44 +             public Promise<Response, NeverThrowsException> handle(Context context, Request request) {
45 +                 Map<String, String> content = new HashMap<>();
46 +                 content.put("applicationName", applicationName);
47 +                 content.put("matchedUri", context.asContext(UriRouterContext.class).getBaseUri());
48 +                 return newResultPromise(new Response(Status.OK).setEntity(content));
49 +             }
50 +         };
51 +     }
52 +
53 +     @Override
54 +     public Factory<Buffer> getBufferFactory() {
55 +         return null;
56 +     }
57 +
58 +     @Override
59 +     public void stop() {
60 +
61 +     }
62 + }
63 + ```
64 +
65 + When run, you can then do requests such as:
66 +
67 + ```
68 + $ curl http://localhost:8080/show/me
69 + {"matchedUri":"","applicationName":"default"}
70 + ```
71 +
72 + ## Describing APIs
73 +
74 + In CHF applications, the [OpenAPI Specification](https://github.com/OAI/OpenAPI-Specification/) can be used to describe
75 + the API being exposed, using the
76 + [models](https://github.com/swagger-api/swagger-core/tree/master/modules/swagger-models) module of
77 + [Swagger](http://swagger.io/).
78 +
79 + In order to expose API Descriptors in this way, you must do two things:
80 + * Instead of implementing the `HttpApplication` interface, use
81 + [`DescribedHttpApplication`](http-core/src/main/java/org/forgerock/http/DescribedHttpApplication.java)
82 + * Instead of implementing `Handler` for your application, implement
83 + [`DescribableHandler`](http-core/src/main/java/org/forgerock/http/handler/DescribableHandler.java)
84 +
85 + The [ExampleHttpApplication](http-examples/http-servlet-example/src/main/java/org/forgerock/http/servlet/example/ExampleHttpApplication.java)
86 + referenced above implements the `DescribedHttpApplication` class, and there is an example handler,
87 + [DescribedOauth2Endpoint](http-examples/http-descriptor-example/src/main/java/org/forgerock/http/example/DescribedOauth2Endpoint.java)
88 + that shows the simple case of how one might expose a static descriptor.
