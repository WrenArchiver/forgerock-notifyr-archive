93c73c50952 RELENG 89: Detach third party license reporting from parent build phase

Since the script that extracts third party licensing information is
never used by the parent POM, it now exists there only as
configuration, rather than being bound to a build phase. Instead, it is
down to the child projects to bind the execution of the code to a build
phase (presumably by activating the 'thirdpartylicensing' profile).

This provides a better mechanism for distributing the code that extracts
the third party license reports (and the configuration of the plugin
that executes it), while ensuring the code is not executed in the parent
POM and allowing the child projects to explicitly specify where the code
should be executed.		 r.smith 09 Mar - 10:25 AM
