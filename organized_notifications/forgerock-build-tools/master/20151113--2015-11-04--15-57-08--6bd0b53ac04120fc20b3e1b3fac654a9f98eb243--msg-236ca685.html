a72e5a37469 Add important comment to avoid cycles and allow automatic releasing

If forgerock-parent 2.0.4 depends on this module (in this version), and
this module depends on forgerock-parent 2.0.4, how can you do a maven release ?

That's a chicken and egg problem.

Here I choose to depend on version n-1 of the parent for simplicity.
The other solution would have been to have **no** parent POM and redefine
properties, plugins, repositories: the bare minimum that we need to
release this module independently.		 04 Nov 2015
