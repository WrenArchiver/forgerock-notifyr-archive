        <p style='margin-top:0;margin-bottom:10px;'>When Internet Explorer 9 attempts to load forgerock-ui (any instance of it, including the mock project), the first time everything works fine. However, subsequent loads of the UI (such as from a refreshed page) will fail; it will never initialize the application. Clearing the cache before refreshing the page will cause it to load again. This appears to have started happening with the upgrade of libraries that were done as part of forgerock-ui 1.1. The main culprit appears to be jQuery, and most in particularly the way in which we are loading jQuery twice - once using a regular script tag, and then once again via requirejs. Having to do this is a hack anyway - the solution will be to finally remove this hack and only load jquery with requirejs (at the same time, we should remove the similar hacks for lodash and backbone).</p>
