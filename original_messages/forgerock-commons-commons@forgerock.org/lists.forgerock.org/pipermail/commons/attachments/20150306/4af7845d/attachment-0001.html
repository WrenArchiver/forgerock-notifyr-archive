                                            <h3 style="margin: 10px 0 0 0; font-size: 16px; line-height: 1.563; margin: 30px 0 0 0"><a name="SummaryofBestPractices" style="color: #3b73af; text-decoration: none"></a>Summary of Best Practices</h3> 
                                            <p style="margin: 10px 0 0 0">The work for this issue was to look at how some popular specifications/frameworks and SaaS providers handle REST API descriptions.</p> 
                                            <p style="margin: 10px 0 0 0">Frameworks/specifications and SaaS provider practices are still evolving, but also seem to be converging in these areas:</p> 
                                            <ul> 
                                                <li>
                                                    Popular frameworks specify API descriptor languages that map REST APIs to HTTP. Big SaaS providers with many REST APIs tend to have their own descriptor languages (Google, LinkedIn, Salesforce.com), which the publicize to some degree. Providers who implement services in Java like LinkedIn and Salesforce.com do seem to use annotations and Javadoc to store descriptor information.
                                                </li> 
                                                <li>
                                                    Descriptor languages tend to lack support for recursive models, polymorphism (though Swagger 2.0 introduces some support), or overloading HTTP methods.
                                                </li> 
                                                <li>
                                                    Both frameworks and vendors use descriptor languages to facilitate documentation/discovery UIs, validation &amp; testing, and SDK generation for languages like JavaScript, PHP, Python, Ruby and platforms like iOS and Android.
                                                </li> 
                                                <li>
                                                    The frameworks and most providers use JSON Schema for JSON resources (XSD for XML), and basic URI templates to describe endpoints.
                                                </li> 
                                                <li>
                                                    Some frameworks include information about authentication in the API descriptor, referencing well-defined security schemes like OAuth 2.0 or HTTP Basic.
                                                </li> 
                                            </ul>
