                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">Requests via REST API with data but no content-type should be refused but are now accepted</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">With fr-rest 2.5.0 (used by openIDM since commit r5496 done for <a href="https://bugster.forgerock.org/jira/browse/OPENIDM-3527" title="Update 3.2.0 to use new forgerock-util 2.0.0-SNAPSHOT dependencies" class="issue-link" data-issue-key="OPENIDM-3527" style="color: #3b73af; text-decoration: none"></a></p>
                                            <del>
                                                OPENIDM-3527
                                            </del>), the behavior of requests on REST API wwith regard to content-type changed.<br /> <a href="http://openidm.forgerock.org/doc/bootstrap/integrators-guide/index.html#rest-content-negotiation" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">Integrator's guide section on &quot;content negociation&quot;</a> mentions that &quot;In most cases, you should include the following header&quot; and &quot;The application/json content type is not needed when a REST call is made with the GET and DELETE methods&quot;. And indeed, so far, header &quot;content-type&quot; was mandatory on PUT and POST requests that contained data sent in the body. <br /> Now with r5496, this header is no more mandatory =&gt; this is a regression 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">With plain-vanilla OpenIDM, here is how to reproduce.<br /> 1) launch OpenIDM<br /> 2) use a REST client to sent a POST that creates a user<br /> Note: I am not using curl (command line) or Postman (Chrome app) because they seem to send default content-type.<br /> So I am using <a href="http://docs.python-requests.org/en/latest/" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">Python requests library</a> here:</p>
