<h3>Log Message</h3>
<pre>Make the test testReadingAuditLogEntry more robust

* Use Files.createTempDirectory to really create a temporary directory.
Before we always ended to use /tmp and thus that was the same file
access.csv that was used and created some randomness in that test.
* Do not reuse the same mocks nor ResourceCaptor : make the test more
readable
* Drop the null entries before comparing the JsonValue for the
assertions.</pre>
