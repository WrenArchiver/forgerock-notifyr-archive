                                        <td class="text-paragraph-pattern-container mobile-resize-text " style="padding: 0px; border-collapse: collapse; padding: 0 0 10px 0"> 
                                            <p style="margin: 10px 0 0 0">Based on a previous discussion in the OpenIDM team that Jake shared with me, Chad and others had seen the need to extend JSON Schema with some additional properties for the UI:</p> 
                                            <blockquote style="margin: 10px 0 0 0; border-left: 1px solid #ccc; color: #707070; margin-left: 19px; padding: 10px 20px"> 
                                                <ul> 
                                                    <li>
                                                        &quot;resourceCollection&quot;: An object specifying the &quot;path&quot; and &quot;query&quot; information of a resourceCollection
                                                    </li> 
                                                    <li>
                                                        &quot;searchable&quot;: A boolean indicating if the field is searchable.
                                                    </li> 
                                                    <li>
                                                        &quot;viewable&quot;: A boolean indicating if the field is viewable.
                                                    </li> 
                                                    <li>
                                                        &quot;label&quot;: A human readable label for the field.
                                                    </li> 
                                                    <li>
                                                        &quot;order&quot;: An array of property names indicating an order of the object's fields.
                                                    </li> 
                                                </ul> 
                                                <p style="margin: 10px 0 0 0">&quot;resourceCollection&quot;... is an attempt at referencing another resource within OpenIDM. Typically it would be another managed object, but it could also be any resource that is in a resourceCollection. The reason for the &quot;Collection&quot; part of the name is that we could also add a property to represent a resourceSingleton which would most likely require additional sub-properties .</p>
                                            </blockquote> 
                                            <p style="margin: 10px 0 0 0">To summarize these requirements, in addition to JSON Schema capabilities, the API descriptor should also have the capability to indicate:</p> 
                                            <ul> 
                                                <li>
                                                    Field ordering
                                                </li> 
                                                <li>
                                                    Human-readable labels for fields
                                                </li> 
                                                <li>
                                                    Whether a field is viewable
                                                </li> 
                                                <li>
                                                    Whether a field is searchable
                                                </li> 
                                                <li>
                                                    The path/query to look up the object referenced by a field, based on its value
                                                </li> 
                                            </ul>
