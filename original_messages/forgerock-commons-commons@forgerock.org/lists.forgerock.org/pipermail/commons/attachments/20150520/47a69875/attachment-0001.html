                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">If an OpenIdConnect token is created with a JwsHeaderBuilder which sets non-string jose header values (e.g. jku, or jwk), its reconstruction will fail with a stack-trace like the ones below. It appears that the JwtSecureHeader.setParameter should try to parse the Object value into the expected type (e.g. JWK) using the appropriate parsing method (e.g. JWK.parse) as part of the reconstruction. In other words, JwtReconstruction will attempt to reconstruct the JwsSecureHeader state without first attempting to marshal the base64-decoded string into the object types expected by the setParameter method. </p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">It appears that this error is never encountered previously because the OpenIdConnect tokens created as part of the OAuth2 provider only set the kid, which is a string, and thus can be marshaled with the current, basic approach.</p>
