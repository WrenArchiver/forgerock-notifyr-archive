        <div class="comment-block" style="background-color:#edf5ff;border:1px solid #dddddd;color:#000000;padding:12px;"><p>Changes to address this bug were checked-in to the 2.x branch.</p>

<p>The changes to address this in trunk are a bit more problematic:<br/>
Please see the changes in <a href="https://bugster.forgerock.org/jira/browse/CREST-184" title="VersionRouter#removeRoute parameter is UriRoute... instead of Route...">CREST-184</a>_trunk.diff in <a href="http://sources.forgerock.org/cru/CMN-13">http://sources.forgerock.org/cru/CMN-13</a>, specifically the blind cast in Router#removeRoute. In trunk, Router delegates to the UriRouter class, which exposes route mutations with the UriRoute class. There is no UriRouter class in the 2.x branch. I see two options:<br/>
1. the blind cast in Router#removeRoute is necessary, and safe (at the moment), because Router delegates to UriRouter, which exposes route mutations using the UriRoute class. Yet if this delegation is altered, the blind cast will likely cause problems.<br/>
2. The UriRouter class must expose its route mutations using the Route interface, not the UriRoute class.</p>

<p>The trunk patch in <a href="https://bugster.forgerock.org/jira/browse/CREST-184" title="VersionRouter#removeRoute parameter is UriRoute... instead of Route...">CREST-184</a> represents option #1, and was chosen mainly because it was less invasive than option #2. This does not mean that it is the correct approach however, but I can't make this decision.</p>
