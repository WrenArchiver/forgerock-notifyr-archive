                                            <p style="margin: 10px 0 0 0">I spent quite a bit of time over the break playing with Swagger. I got a prototype working. It's hand-coded, not generated, but hand coding JSON could still be useful if no progress is made towards a generation scenario. </p> 
                                            <p style="margin: 10px 0 0 0">I agree with Matt's comments above that Swagger doesn't exactly fit our requirements but perhaps in a manual implementation we could work around the problems Matt has identified. I'm pretty sure we could handle the first issue in the Swagger framework; we could probably handle the second one with examples; the third one I'm not sure, as I'm not familiar with the problem.</p> 
                                            <p style="margin: 10px 0 0 0">An additional issue (especially if we were to hand code JSON) is the lack of proper modularity in the Swagger JSON schema. It doesn't exist at all in Swagger 1.0 and there are only some limited possibilities in Swagger 2.0 (and they are broken in Swagger UI). I prototyped (using a simple bash script) an implementation of modularity with Swagger 2.0 that could be evolved a little bit to meet our needs.</p> 
                                            <p style="margin: 10px 0 0 0">Even with the limitations and the work involved in hand-coding JSON, the Swagger UI framework provides a lot better experience for the end user than what we have now. Using it would give us a consistent look to our API docs, would make it possible for engineers to doc their APIs as they developed them, and would make revising and QA'ing the API doc a more manageable experience than it is today with our text-based docs. </p> 
                                            <p style="margin: 10px 0 0 0">Another exciting possibility available with Swagger UI is the ability to make a running instance (or an Amazon AMI) available that lets users actually run the REST API commands from the doc. While it would be an effort to put that together, it would be a terrific addition to our doc set.</p>
