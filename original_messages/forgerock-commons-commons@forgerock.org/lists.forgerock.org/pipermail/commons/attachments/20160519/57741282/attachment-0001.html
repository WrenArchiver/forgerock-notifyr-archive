                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">There is a mistaken understanding coded into the current implementation of the ObjectUtil module, regarding the treatment of array values. The original assumption was that when JSON values as part of forgerock-rest resources contain multivalued properties (</p> {&quot;roles&quot;: [&quot;admin&quot;,&quot;user&quot;,&quot;sales&quot;]} 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">), the items within have an order which is preserved and consistent from one request to the next. Based on this assumption, the code in ObjectUtil leverages the specific position of the item in the array to support patch operations. For example, given a JSON value like so:</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
{ <span class="code-quote" style="color: #009100">&quot;favorites&quot;</span>:[{<span class="code-quote" style="color: #009100">&quot;type&quot;</span>:<span class="code-quote" style="color: #009100">&quot;candy&quot;</span>,<span class="code-quote" style="color: #009100">&quot;flavor&quot;</span>:<span class="code-quote" style="color: #009100">&quot;lemon&quot;</span>},{<span class="code-quote" style="color: #009100">&quot;type&quot;</span>:<span class="code-quote" style="color: #009100">&quot;candy&quot;</span>,<span class="code-quote" style="color: #009100">&quot;flavor&quot;</span>:<span class="code-quote" style="color: #009100">&quot;cherry&quot;</span>}] }
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">And you want to change it to be this value:</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">

{ <span class="code-quote" style="color: #009100">&quot;favorites&quot;</span>:[{<span class="code-quote" style="color: #009100">&quot;type&quot;</span>:<span class="code-quote" style="color: #009100">&quot;candy&quot;</span>,<span class="code-quote" style="color: #009100">&quot;flavor&quot;</span>:<span class="code-quote" style="color: #009100">&quot;lemon&quot;</span>}] }
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">The present code would create a patch operation like so:</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
{<span class="code-quote" style="color: #009100">&quot;operation&quot;</span>:<span class="code-quote" style="color: #009100">&quot;remove&quot;</span>, <span class="code-quote" style="color: #009100">&quot;field&quot;</span>: <span class="code-quote" style="color: #009100">&quot;/favorites/1&quot;</span>}

                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">This assumption is true in some implementations (notably, OpenIDM) but false in other implementations (notably, OpenDJ). In both OpenIDM and OpenDJ, a patch operation to achieve the above is like so:</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">

{ <span class="code-quote" style="color: #009100">&quot;operation&quot;</span> : <span class="code-quote" style="color: #009100">&quot;remove&quot;</span>, <span class="code-quote" style="color: #009100">&quot;field&quot;</span> : <span class="code-quote" style="color: #009100">&quot;/favorites&quot;</span>, <span class="code-quote" style="color: #009100">&quot;value&quot;</span>: {<span class="code-quote" style="color: #009100">&quot;type&quot;</span>:<span class="code-quote" style="color: #009100">&quot;candy&quot;</span>,<span class="code-quote" style="color: #009100">&quot;flavor&quot;</span>:<span class="code-quote" style="color: #009100">&quot;cherry&quot;</span>} }
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">Note the position of the value to remove isn't included; instead the value of the item is used to identify which you want to remove. This has several implications:</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">1) The position of the item in the list is not relevant<br /> 2) The values within the list must be unique</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">Essentially, to work with the forgerock-rest patch implementation, we must interpret JSON lists as sets, rather than as ordered arrays. Given that this is the target environment for our patch operations, it is necessary to change the behavior to align with this understanding.</p>
