                                            <p style="margin: 10px 0 0 0">Here is an idea which might prevent such deadlock (producer start before consumer is registered): The idea is to delay the processing of the query until all required listener are registered:</p> 
                                            <p style="margin: 10px 0 0 0">To do so we could hide handleResult()/handleException() and replace those by PromiseImpl.create(Callable&lt;V&gt; publisher);</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
Promise&lt;Response, NeverThrowsException&gt; handle(Context context, Request request)
{
  <span class="code-keyword" style="color: #000091">return</span> PromiseImpl.create(<span class="code-keyword" style="color: #000091">new</span> Callable&lt;Response&gt;() {
     <span class="code-keyword" style="color: #000091">public</span> Response call() <span class="code-keyword" style="color: #000091">throws</span> Exception {
        <span class="code-comment" style="color: #808080">// Do something synchronous
</span>     }
  });
}
</pre> 
                                                </div>
                                            </div> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
Promise&lt;Response, NeverThrowsException&gt; handle(Context context, Request request)
{
  <span class="code-keyword" style="color: #000091">return</span> PromiseImpl.create(<span class="code-keyword" style="color: #000091">new</span> Callable&lt;Promise&lt;Response&gt;&gt;() {
     <span class="code-keyword" style="color: #000091">public</span> Promise&lt;Response&gt; call() <span class="code-keyword" style="color: #000091">throws</span> Exception {
        <span class="code-comment" style="color: #808080">// Do something asynchronous
</span>     }
  });
}
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin: 10px 0 0 0">This allow the caller code to register all the required listener before the processing start:</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
<span class="code-comment" style="color: #808080">// Synchronous processing: Here, .get() start the request handling.
</span>Response theSynchronousResponse = whateverHandler.handle(context, request).then(...).thenAsync(...).get(); 

<span class="code-comment" style="color: #808080">// Asynchronous processing: This <span class="code-keyword" style="color: #000091; color: #808080">return</span> void (or a Future ?). The idea is that the Promise cannot be chained anymore after startProcess.
</span>yetAnotherHandler.handle(context, request)
                         .then(...).thenAsync(...)
                         .startProcess(<span class="code-keyword" style="color: #000091">new</span> ResultHandler&lt;Response&gt;() { 
                              ...
                              <span class="code-comment" style="color: #808080">// Do something with the response
</span>                              ...
                         }, <span class="code-keyword" style="color: #000091">new</span> ExceptionHandler&lt;Exception&gt;() { ... }); 
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin: 10px 0 0 0">Some notable side-effects:</p> 
                                            <ul class="alternate" type="square"> 
                                                <li>
                                                    The processing is delayed until the full Promise chain is built.
                                                </li> 
                                                <li>
                                                    If you forget to .get() or .startProcess() no processing will be performed (btw, we could catch those programming error using watchdog).
                                                </li> 
                                            </ul> 
                                            <p style="margin: 10px 0 0 0">I can write a quick &amp; dirty PoC of such Promise impl if you think this makes sense ?</p>
