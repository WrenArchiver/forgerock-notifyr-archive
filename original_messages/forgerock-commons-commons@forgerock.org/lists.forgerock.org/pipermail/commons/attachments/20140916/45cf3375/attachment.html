        <p style='margin-top:0;margin-bottom:10px;'>Currently when your session expires or you attempt an action which is not authorized, the EVENT_UNAUTHORIZED process gets triggered. The logic within this event is intended to detect what type of request produced the original error; if it is a GET, then we just redirect to the login page (while saving the current url to take us back after login). If it is not a GET, then we show the local implementation of "LoginDialog". The problem with the current approach is that it resets the state of the conf.loggedUser (amongst other things) before the login dialog is shown, which may not be appropriate depending on the method for reauthentication found in the dialog.</p>

<p style='margin-top:0;margin-bottom:10px;'>The proper behavior for this event should be that GET requests call sessionManager.logout prior to redirecting to the LoginView, and that non-GET requests merely display the LoginDialog.</p>
