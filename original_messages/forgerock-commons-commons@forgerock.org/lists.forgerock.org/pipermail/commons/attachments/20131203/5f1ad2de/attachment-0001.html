+  &lt;section name=&quot;About the Sample App&quot;&gt;
+
+   &lt;p&gt;
+    &lt;a href=&quot;http://forgerock.com/products/open-identity-stack/openam/&quot;
+    &gt;OpenAM services&lt;/a&gt; make it easy to add access management
+    to modern mobile applications.
+   &lt;/p&gt;
+
+   &lt;p&gt;
+    OpenAM OAuth 2.0 iOS Sample App demonstrates use of OpenAM server's
+    standard &lt;a href=&quot;http://docs.forgerock.org/en/openam/11.0.0/dev-guide/index/chap-rest.html#rest-api-oauth2&quot;
+    &gt;OAuth 2.0 REST API&lt;/a&gt; to authorize OAuth 2.0 client access.
+   &lt;/p&gt;
+
+   &lt;p&gt;
+    For more information on OpenAM's support for OAuth 2.0,
+    read the &lt;i&gt;Administration Guide&lt;/i&gt; chapter on
+    &lt;a href=&quot;http://docs.forgerock.org/en/openam/11.0.0/admin-guide/index/chap-oauth2.html&quot;
+    &gt;&lt;i&gt;Managing OAuth 2.0 Authorization&lt;/i&gt;&lt;/a&gt;.
+   &lt;/p&gt;
+
+   &lt;p&gt;
+    For more information on OpenAM's support for mobile applications,
+    read the &lt;i&gt;Administration Guide&lt;/i&gt; chapter on
+    &lt;a href=&quot;http://docs.forgerock.org/en/openam/11.0.0/admin-guide/index/chap-mobile.html&quot;
+    &gt;&lt;i&gt;Working with Mobile Devices &amp;amp; Applications&lt;/i&gt;&lt;/a&gt;.
+   &lt;/p&gt;
+
+  &lt;/section&gt;
+
+  &lt;section name=&quot;To Prepare OpenAM Server&quot;&gt;
+
+   &lt;p&gt;
+    Before you try the sample application, install and configure OpenAM server
+    as described in the &lt;a href=&quot;http://docs.forgerock.org/en/openam/11.0.0/install-guide/&quot;
+    &gt;&lt;i&gt;OpenAM Installation Guide&lt;/i&gt;&lt;/a&gt;, and then prepare OpenAM
+    for use with the sample application.
+   &lt;/p&gt;
+
+   &lt;p&gt;
+    &lt;b&gt;Important&lt;/b&gt; For the initial version of the sample,
+    the cookie domain must be &lt;code&gt;.forgerock.com&lt;/code&gt;,
+    the OpenAM administrator &lt;code&gt;amadmin&lt;/code&gt;
+    must have password &lt;code&gt;cangetin&lt;/code&gt;,
+    and the SSO Token cookie must have the default name,
+    &lt;code&gt;iPlanetDirectoryPro&lt;/code&gt;.
+   &lt;/p&gt;
+
+   &lt;p&gt;
+    &lt;b&gt;Tip&lt;/b&gt; Add an alias to the localhost address
+    such as &lt;code&gt;oauth2.forgerock.com&lt;/code&gt;
+    in your &lt;code&gt;/etc/hosts&lt;/code&gt; file
+    to fake the fully-qualified domain name before configuring OpenAM.
+   &lt;/p&gt;
+
+   &lt;p&gt;
+    After initial installation and configuration of OpenAM,
+    follow these steps to prepare OpenAM for use with the sample application.
+   &lt;/p&gt;
+
+   &lt;ol&gt;
+
+    &lt;li&gt;
+     &lt;p&gt;
+      In OpenAM Console &gt; Common Tasks, click Configure OAuth2,
+      and then click Create in the Configure OAuth2 page
+      to configure an OAuth 2.0 Authorization Server for the realm /.
+     &lt;/p&gt;
+    &lt;/li&gt;
+
+
+    &lt;li&gt;
+     &lt;p&gt;
+      In OpenAM Console &gt; Access Control &gt; / (Top Level Realm) &gt; Agents
+      &gt; OAuth 2.0 Client, click New,
+      and then use the New OAuth 2.0 Client page
+      to register an OAuth 2.0 client
+      with name &lt;code&gt;MyApp&lt;/code&gt; and password &lt;code&gt;password&lt;/code&gt;.
+     &lt;/p&gt;
+
+     &lt;p&gt;
+      After you register the client, click the link to the client profile
+      to set the Redirection URI to &lt;code&gt;http://www.example.com/&lt;/code&gt;
+      and the Scopes to &lt;code&gt;cn&lt;/code&gt; and &lt;code&gt;mail&lt;/code&gt;,
+      and then click Save.
+     &lt;/p&gt;
+    &lt;/li&gt;
+
+   &lt;/ol&gt;
+
+  &lt;/section&gt;
+
+  &lt;section name=&quot;To Build the Sample App on Mac OS X&quot;&gt;
+
+   &lt;p&gt;
+    You can build the sample on OS X 10.8 or later with Xcode installed.
+   &lt;/p&gt;
+
+   &lt;ol&gt;
+
+     &lt;li&gt;
+
+      &lt;p&gt;
+       In Terminal, check out the sample application source code.
+      &lt;/p&gt;
+
+      &lt;pre&gt;$ svn co https://svn.forgerock.org/commons/samples/mobile/ios&lt;/pre&gt;
+
+      &lt;p&gt;
+       This subversion command checks out iOS sample application code into
+       an &lt;code&gt;ios&lt;/code&gt; folder in the current working directory.
+       It also checks out the REST SDK framework code
+       that you need in order to build the sample application.
+      &lt;/p&gt;
+     &lt;/li&gt;
+
+     &lt;li&gt;
+
+      &lt;p&gt;
+       Build the REST SDK framework that the sample application depends on.
+      &lt;/p&gt;
+
+      &lt;pre&gt;$ cd ios/openam-ios-rest-sdk/trunk/
+$ rake&lt;/pre&gt;
+
+      &lt;p&gt;
+       When you build with &lt;code&gt;rake&lt;/code&gt; on the command line,
+       the framework is copied to your Frameworks folder,
+       &lt;code&gt;~/Library/Frameworks/openam-ios-rest-sdk.framework&lt;/code&gt;.
+      &lt;/p&gt;
+
+     &lt;/li&gt;
+
+     &lt;li&gt;
+
+      &lt;p&gt;
+       Build the sample project in Xcode.
+      &lt;/p&gt;
+
+      &lt;p&gt;
+       You can open the project either from Finder, or from the command line.
+      &lt;/p&gt;
+
+      &lt;pre&gt;$ cd ../../openam-ios-oauth2-sample-app/trunk/
+$ open openam-ios-oauth2-sample-app.xcodeproj&lt;/pre&gt;
+
+     &lt;/li&gt;
+
+     &lt;li&gt;
+
+      &lt;p&gt;
+       Click the play button in Xcode to build and then run the current scheme.
+       After the sample builds, Xcode launches the sample in the simulator.
+      &lt;/p&gt;
+
+    &lt;/li&gt;
+
+   &lt;/ol&gt;
+
+   &lt;p&gt;
+    If you have successfully arrived at this point,
+    then OpenAM can function as an OAuth 2.0 Authorization Server,
+    and the client application is registered as &lt;code&gt;MyApp&lt;/code&gt;.
+    The client application is also running in an iOS device simulator,
+    waiting to be configured.
+   &lt;/p&gt;
+
+  &lt;/section&gt;
+

+   &lt;p&gt;

+    in an iOS device simulator, and you can try it with OpenAM.
+   &lt;/p&gt;
+
+   &lt;ol&gt;
+    &lt;li&gt;
+
+     &lt;p&gt;
+      The initial screen in the simulator lets you configure the client.
+     &lt;/p&gt;
+

+
+      &lt;li&gt;
+       Edit the OAuth2 client settings to correspond

+      for the client application.
