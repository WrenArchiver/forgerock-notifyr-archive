        <div class="comment-block" style="background-color:#edf5ff;border:1px solid #dddddd;color:#000000;padding:12px;"><p>I would like to propose a slight variation to the given suggested behavior.</p>

<p>Given a request like so:</p>
<div class="code panel" style="border-width: 1px;"><div class="codeContent panelContent">
<pre class="code-java">POST /users?_action=create  HTTP/1.1
Host: example.com
Accept: application/json
Content-Type: multipart/form-data; boundary=kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL
Content-Length: ...

--kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL
Content-Disposition: form-data; name=<span class="code-quote">"json"</span>
Content-Type: application/json; charset=US-ASCII
Content-Transfer-Encoding: 8bit

{
  <span class="code-quote">"uid"</span>             : <span class="code-quote">"alice"</span>,
  <span class="code-quote">"email"</span>           : <span class="code-quote">"alice@example.com"</span>,
  <span class="code-quote">"profilePhoto"</span>    : <span class="code-quote">"${profilePhoto.content}"</span>,
  <span class="code-quote">"profilePhotoType"</span>: <span class="code-quote">"${profilePhoto.mimetype}"</span>,
  <span class="code-quote">"profilePhotoName"</span>: <span class="code-quote">"${profilePhoto.filename}"</span>
}
--kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL
Content-Disposition: form-data; name=<span class="code-quote">"profilePhoto1"</span>; filename=<span class="code-quote">"HappyJSON.jpg"</span>
Content-Type: image/jpeg
Content-Transfer-Encoding: binary

*** binary image data ***

--kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL--</pre>
</div></div>


<p>I would expect that the method handling "CREATE" for /users (in this example) would be handed a JSON value that looked like so:</p>

<div class="code panel" style="border-width: 1px;"><div class="codeContent panelContent">
<pre class="code-java">{
  <span class="code-quote">"uid"</span>             : <span class="code-quote">"alice"</span>,
  <span class="code-quote">"email"</span>           : <span class="code-quote">"alice@example.com"</span>,
  <span class="code-quote">"profilePhoto"</span>    : <span class="code-quote">"...base 64 encoded image data..."</span>,
  <span class="code-quote">"profilePhotoType"</span>: <span class="code-quote">"image/jpeg"</span>,
  <span class="code-quote">"profilePhotoName"</span>: <span class="code-quote">"HappyJSON.jpg"</span>
}</pre>
</div></div>

<p>Using the "${formPart.data}" style as a placeholder for the literal value pulled out of the POST body will allow us a lot of flexibility in how and where the various values we may be interested in are represented. As another example, consider this:</p>

<div class="code panel" style="border-width: 1px;"><div class="codeContent panelContent">
<pre class="code-java">PATCH /users/alice
Content-Type: multipart/form-data; boundary=kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL
--kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL
Content-Disposition: form-data; name=<span class="code-quote">"json"</span>
Content-Type: application/json; charset=US-ASCII
Content-Transfer-Encoding: 8bit
[
 { <span class="code-quote">"field"</span>:<span class="code-quote">"profilePhoto"</span>,     <span class="code-quote">"value"</span>: <span class="code-quote">"${profilePhoto.content}"</span>,  <span class="code-quote">"action"</span>: <span class="code-quote">"replace"</span> },
 { <span class="code-quote">"field"</span>:<span class="code-quote">"profilePhotoType"</span>, <span class="code-quote">"value"</span>: <span class="code-quote">"${profilePhoto.mimetype}"</span>, <span class="code-quote">"action"</span>: <span class="code-quote">"replace"</span> },
 { <span class="code-quote">"field"</span>:<span class="code-quote">"profilePhotoName"</span>, <span class="code-quote">"value"</span>: <span class="code-quote">"${profilePhoto.filename}"</span>, <span class="code-quote">"action"</span>: <span class="code-quote">"replace"</span> }
]
--kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL
Content-Disposition: form-data; name=<span class="code-quote">"profilePhoto1"</span>; filename=<span class="code-quote">"HappyJSON.jpg"</span>
Content-Type: image/jpeg
Content-Transfer-Encoding: binary

*** binary image data ***

--kwkIqb-fsdbtcNpB4dJ_Xqf1-3b0Hp_VF9D0vsgL--</pre>
</div></div>

<p>The point with this second example is that you may want to refer to your bits of form data at unpredictable places within your JSON body.</p></div>
