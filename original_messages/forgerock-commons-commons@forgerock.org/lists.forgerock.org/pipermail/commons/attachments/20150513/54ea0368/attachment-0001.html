<h3>Log Message</h3>
<pre>CHF-34 (CMN-128) Use Promise&lt;Response, NeverThrowsException&gt;

 * `Response` is a raw response message usually obtained from a remote Http server
   It's perfectly ok to have a response that do not have a `2xx` status code.
 * `ResponseException` is more a processing exception (something gone wrong
   on the client side, before sending the `Request` or after receiving the `Response`).
   Notice that this exception is provided as a
   *convenience* for Filter/Handler implementers
 * Removed `NotFoundException` because CHF `UriRouter` now generates the
   appropriate `404` response
 * Updated javadoc accordingly</pre>
