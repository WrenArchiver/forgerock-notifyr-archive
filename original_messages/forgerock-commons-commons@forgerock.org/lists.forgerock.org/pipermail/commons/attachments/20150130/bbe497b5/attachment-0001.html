                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">Firstly there's no support for setting response cookies. In addition, response cookies must be encoded using using separate header lines, rather than combined as a single comma separated line.</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">RequestCookies are immutable, so it is not possible to add and remove cookies very easily. It would be nice if it were possible, since cookie manipulation is quite complex.</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">In addition, like Headers, individual Cookies are not synced with the underlying message. I think this is ok, except in a scripting context where as developer may be tempted to do (dumb example):</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
request.cookies.foo.httpOnly = <span class="code-keyword" style="color: #000091">true</span>
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">Is this a use case we want to support? Are we likely to transform cookies in place during processing? We may want to support certain transformations of the cookie value, e.g cryptographic transformations (there is an RFE for this in OpenIG). In this case I think it's ok to retrieve the cookie, transform its content, and then replace the cookie in the request/response. In other words, I don't think individual cookie objects should be synced with the underlying message, but I do think the cookie map should be mutable in order to support replacement of cookies.</p>
