                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">I see two options:</p> 
                                            <ul> 
                                                <li>
                                                    we always include the _id and _rev field in responses regardless of the field filtering
                                                </li> 
                                                <li>
                                                    we construct the ResourceResponse with a null id and rev, potentially triggering NPEs.
                                                </li> 
                                            </ul> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">I think taking the second approach could cause problems if we ever want to call out to remote CREST endpoints, so my preference is for the first option.</p>
