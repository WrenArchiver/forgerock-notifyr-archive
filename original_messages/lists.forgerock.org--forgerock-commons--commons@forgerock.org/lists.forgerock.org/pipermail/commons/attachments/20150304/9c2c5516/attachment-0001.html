                                            <p style="margin: 10px 0 0 0">A summary of observations about popular frameworks and specifications follows.</p> 
                                            <h3 style="margin: 10px 0 0 0; font-size: 16px; line-height: 1.563; margin: 30px 0 0 0"><a name="APIBlueprint" style="color: #3b73af; text-decoration: none"></a>API Blueprint</h3> 
                                            <p style="margin: 10px 0 0 0"><a href="https://apiblueprint.org" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">API Blueprint</a> is a language for describing REST APIs, based on and using Markdown for documentation.</p> 
                                            <p style="margin: 10px 0 0 0">Advantages</p> 
                                            <ul> 
                                                <li>
                                                    Human-readable descriptions
                                                </li> 
                                                <li>
                                                    Support for JSON Schema in defining payloads
                                                </li> 
                                                <li>
                                                    Support for testing and validation
                                                </li> 
                                            </ul> 
                                            <p style="margin: 10px 0 0 0">Disadvantages</p> 
                                            <ul> 
                                                <li>
                                                    Missing inheritance, polymorphism
                                                </li> 
                                                <li>
                                                    No Java bindings
                                                </li> 
                                                <li>
                                                    No method overloading
                                                </li> 
                                                <li>
                                                    No recursive data models
                                                </li> 
                                                <li>
                                                    Seems geared for Apiary's commercial offering
                                                </li> 
                                                <li>
                                                    Top-down language description
                                                </li> 
                                            </ul> 
                                            <p style="margin: 10px 0 0 0">Find the specification <a href="https://github.com/apiaryio/api-blueprint/blob/master/API%20Blueprint%20Specification.md" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">on Github</a>.</p> 
                                            <h3 style="margin: 10px 0 0 0; font-size: 16px; line-height: 1.563; margin: 30px 0 0 0"><a name="JSONSchema%26HyperSchema" style="color: #3b73af; text-decoration: none"></a>JSON Schema &amp; Hyper-Schema</h3> 
                                            <p style="margin: 10px 0 0 0">These are draft specs only. Tools are becoming available separately.</p> 
                                            <p style="margin: 10px 0 0 0">JSON Schema seems to be the way things are going for describing JSON resources, with support for composition and inheritance, basic types and constraints on types (including regular expression matching for string values).</p> 
                                            <p style="margin: 10px 0 0 0">JSON Hyper-Schema adds keywords to JSON Schema to define hypermedia (e.g. non-JSON content of other MIME types) and hyperlinks between JSON resources. Link descriptions specify the link target location and the relationship between the current resource and the link target. Link descriptions can also indicate the expected schema or media type of the link target.</p> 
                                            <h3 style="margin: 10px 0 0 0; font-size: 16px; line-height: 1.563; margin: 30px 0 0 0"><a name="RAML" style="color: #3b73af; text-decoration: none"></a>RAML</h3> 
                                            <p style="margin: 10px 0 0 0"><a href="http://raml.org" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">RESTful API Modeling Language (RAML)</a> describes REST APIs that deal with JSON or XML resources. Descriptions can be formatted in Markdown.</p> 
                                            <p style="margin: 10px 0 0 0">Advantages</p> 
                                            <ul> 
                                                <li>
                                                    Human-readable descriptions
                                                </li> 
                                                <li>
                                                    Interactive API Console (though a bit too rough for developer docs)
                                                </li> 
                                                <li>
                                                    Reference for common security schemes
                                                </li> 
                                                <li>
                                                    Support for JSON Schema and XSDs in defining payloads
                                                </li> 
                                            </ul> 
                                            <p style="margin: 10px 0 0 0">Disadvantages</p> 
                                            <ul> 
                                                <li>
                                                    Less mature, including tools
                                                </li> 
                                                <li>
                                                    Missing inheritance, polymorphism
                                                </li> 
                                                <li>
                                                    No Java bindings
                                                </li> 
                                                <li>
                                                    No method overloading
                                                </li> 
                                                <li>
                                                    No recursive data models
                                                </li> 
                                                <li>
                                                    Top-down language description
                                                </li> 
                                            </ul> 
                                            <h3 style="margin: 10px 0 0 0; font-size: 16px; line-height: 1.563; margin: 30px 0 0 0"><a name="Swagger" style="color: #3b73af; text-decoration: none"></a>Swagger</h3> 
                                            <p style="margin: 10px 0 0 0"><a href="http://swagger.io/" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">Swagger</a> is a specification with tools for describing APIs, documenting them, integrating them with services, and generating SDKs for specific languages.<br /> The version under consideration here is the <a href="https://github.com/swagger-api/swagger-spec/blob/master/versions/2.0.md" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">Swagger 2.0 specification</a>, released in September 2014.</p> 
                                            <p style="margin: 10px 0 0 0">Advantages</p> 
                                            <ul> 
                                                <li>
                                                    Bottom-up API description
                                                </li> 
                                                <li>
                                                    Fit with Java ecosystem (though mostly Scala, perhaps not easy to extend in Java)
                                                </li> 
                                                <li>
                                                    Interactive UI tool available
                                                </li> 
                                                <li>
                                                    Reference for common security schemes (e.g. &quot;basic&quot;, &quot;oauth2&quot;)
                                                </li> 
                                                <li>
                                                    Some polymorphism (&quot;discriminator&quot; field in schema objects, though not in inline definitions)
                                                </li> 
                                                <li>
                                                    Subsets JSON Schema draft 4
                                                </li> 
                                                <li>
                                                    Support for XML resources, too
                                                </li> 
                                            </ul> 
                                            <p style="margin: 10px 0 0 0">Disadvantages</p> 
                                            <ul> 
                                                <li>
                                                    No method overloading
                                                </li> 
                                                <li>
                                                    No recursive data models
                                                </li> 
                                                <li>
                                                    Resource representation in interactive UI
                                                </li> 
                                            </ul> 
                                            <p style="margin: 10px 0 0 0">An <a href="https://docs.google.com/a/forgerock.com/document/d/1zsqZfs9qObA0Gze3ahWVEy7YiAQezsyy8WaQ6cWFWtM" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">internal PoC</a> showed that using Swagger UI for documenting REST APIs for which we do not yet have a descriptor could be a workaround, though there are bugs. (It could well be better to hand-write the Swagger description, working around limitations, than to hand-write even more static documentation for such APIs.)</p> 
                                            <h3 style="margin: 10px 0 0 0; font-size: 16px; line-height: 1.563; margin: 30px 0 0 0"><a name="Other" style="color: #3b73af; text-decoration: none"></a>Other</h3> 
                                            <p style="margin: 10px 0 0 0"><a href="https://github.com/mikekelly/hal_specification/blob/master/hal_specification.md" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">Hypertext Application Language (HAL)</a> defines a mechanism for hyperlinking resources in similar fashion to JSON Hyper-Schema.</p> 
                                            <p style="margin: 10px 0 0 0"><a href="http://tools.ietf.org/html/rfc6570" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">URI Templates</a> offer a little regex language for URIs.</p> 
                                            <p style="margin: 10px 0 0 0">For example, the following describes a path with a recursive <em>realm</em> parameter.</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
/json{/realm*}/authenticate
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin: 10px 0 0 0">Current implementations do not appear to support this and other advanced features of the RFC yet, however, though they do support basic (at least Level 1) templates.</p>
