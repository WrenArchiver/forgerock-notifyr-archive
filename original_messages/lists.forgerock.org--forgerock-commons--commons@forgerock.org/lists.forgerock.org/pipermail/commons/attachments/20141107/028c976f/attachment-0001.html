<h3>Log Message</h3>
<pre>CREST-192 CMN39: Fix various resource name parsing issues

* delegate all responsibilities of resource name parsing, trimming, etc to the ResourceName class
* ensure that the resource ID passed to PUT/create requests is URL path decoded
* catch resource name parsing errors and throw them as 400 errors instead of 500s
* add ResourceName based constructors to the Requests class in order to avoid unnecessary String/ResourceName conversions in the Servlet. This also allowed me to remove more brittle path manipulation code.</pre>
