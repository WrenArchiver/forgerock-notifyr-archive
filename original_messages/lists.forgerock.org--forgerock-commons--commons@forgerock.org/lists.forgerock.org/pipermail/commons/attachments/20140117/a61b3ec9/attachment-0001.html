<h3>Log Message</h3>
<pre>Fix CREST-132: Example using create with HTTP PUT fails with HTTP 400

* reviewed and tested by Mark
* fixed PUT path parsing in Servlet adapter to handle root collections, e.g. where &quot;/&quot; is the collection and &quot;/{id}&quot; is a resource in the collection. Previously, it only handled non-root collections, e.g. of the form &quot;/{collection}/{id}&quot;
* fixed Router to handle root collections. It was assuming that the remaining portion of the resource name is always preceded by a &quot;/&quot;. This is not the case for empty (root) URI templates.</pre>
