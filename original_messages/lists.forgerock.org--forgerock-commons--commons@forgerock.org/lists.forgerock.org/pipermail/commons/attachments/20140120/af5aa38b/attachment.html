        <p style='margin-top:0;margin-bottom:10px;'>Given a query filter string like so:</p>

<div class="code panel" style="border-width: 1px;"><div class="codeContent panelContent">
<pre class="code-java">objectGUID eq <span class="code-quote">"&lt;GUID=4d2c5404f896724e961c86da1645fb0b&gt;"</span> or objectGUID eq <span class="code-quote">"&lt;GUID=833e445409304a48b950d64dcc18ed6b&gt;"</span> or objectGUID eq <span class="code-quote">"&lt;GUID=378602f257d75744b519f415407896f7&gt;"</span></pre>
</div></div>

<p style='margin-top:0;margin-bottom:10px;'>We should be able to parse this using normal order of operation rules for the operators such as "or". In this case, the above should be parsed to be equivalent to this:</p>

<div class="code panel" style="border-width: 1px;"><div class="codeContent panelContent">
<pre class="code-java">(objectGUID eq <span class="code-quote">"&lt;GUID=4d2c5404f896724e961c86da1645fb0b&gt;"</span> or objectGUID eq <span class="code-quote">"&lt;GUID=833e445409304a48b950d64dcc18ed6b&gt;"</span>) or objectGUID eq <span class="code-quote">"&lt;GUID=378602f257d75744b519f415407896f7&gt;"</span></pre>
</div></div>

<p style='margin-top:0;margin-bottom:10px;'>Being required to provide the exact parsing order via explicit parenthesis grouping is tedious, and it would be much nicer if we didn't have to do so.</p>
