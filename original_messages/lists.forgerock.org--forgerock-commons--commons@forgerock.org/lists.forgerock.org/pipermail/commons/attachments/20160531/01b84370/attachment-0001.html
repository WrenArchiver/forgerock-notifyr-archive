                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">When clients of CHF, such as OpenIG, are accessing remote clients that trigger a large response, such as a file transfer, OutOfMemoryErrors can be thrown due to the way the current implementation makes use of memory buffers to capture the response.</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0"><a href="http://hc.apache.org/httpcomponents-core-ga/httpcore-nio/apidocs/org/apache/http/nio/protocol/BasicAsyncResponseConsumer.html" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">http://hc.apache.org/httpcomponents-core-ga/httpcore-nio/apidocs/org/apache/http/nio/protocol/BasicAsyncResponseConsumer.html</a></p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">

                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
Basic implementation of HttpAsyncResponseConsumer. Please note that <span class="code-keyword" style="color: #000091">this</span> consumer buffers response content in memory and should be used <span class="code-keyword" style="color: #000091">for</span> relatively small response messages.
</pre> 
                                                </div>
                                            </div> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">It would be nice if when creating a new HttpClientHandler that there was an option to specify that the file-backed implementation was used instead - <a href="https://hc.apache.org/httpcomponents-asyncclient-dev/httpasyncclient/apidocs/org/apache/http/nio/client/methods/ZeroCopyConsumer.html" class="external-link" rel="nofollow" style="color: #3b73af; text-decoration: none">https://hc.apache.org/httpcomponents-asyncclient-dev/httpasyncclient/apidocs/org/apache/http/nio/client/methods/ZeroCopyConsumer.html</a></p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">
                                                <div class="codeContent panelContent" style="padding: 9px 12px"> 
                                                    <pre class="code-java" style="margin: 10px 0 0 0; max-height: 30em; overflow: auto; white-space: pre-wrap; word-wrap: normal">
HttpAsyncResponseConsumer implementation that streams content entity enclosed in an HTTP response directly into a file without an intermediate in-memory buffer.
This consumer can be useful <span class="code-keyword" style="color: #000091">for</span> file downloads.
</pre> 
                                                </div>
