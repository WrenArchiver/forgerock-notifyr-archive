    <td style="color:#000000;font-family:Arial,FreeSans,Helvetica,sans-serif;font-size:12px;padding:0 0 10px 0;width:100%;">
        <p style='margin-top:0;margin-bottom:10px;'>In the recent Context refactor, a <tt>Context</tt> could implement <tt>ClientContext</tt> and program logic could establish reasoning about a client's position in a context chain using the <tt>ClientContext#hasProtocol</tt> method.  Thus, an <tt>HttpContext</tt> was said to be "external" if </p>
<div class="code panel" style="border-width: 1px;"><div class="codeContent panelContent">
<pre class="code-java">context.asContext(ClientContext.class).hasProtocol(Protocol.valueOf(<span class="code-quote">"http"</span>)) == <span class="code-keyword">true</span></pre>
</div></div>
<p style='margin-top:0;margin-bottom:10px;'>While likely that HttpContext is the only Context considered external, the lack of a unifying implementation of <tt>isExternal()</tt> stack-wide could lead to individual expressions of what it mean to be an "external" calling context.</p>

<p style='margin-top:0;margin-bottom:10px;'>This task recommends replacing the admittedly richer <tt>Protocol</tt> system with a simple ClientContext interface:</p>
<div class="code panel" style="border-width: 1px;"><div class="codeContent panelContent">
<pre class="code-java"><span class="code-keyword">public</span> <span class="code-keyword">interface</span> ClientContext <span class="code-keyword">extends</span> Context {
    /**
     * Return whether <span class="code-keyword">this</span> Context represents an external caller.
     *
     * @<span class="code-keyword">return</span> whether <span class="code-keyword">this</span> context represents an external caller.
     */
    <span class="code-object">boolean</span> isExternal();
}</pre>
</div></div>

<p style='margin-top:0;margin-bottom:10px;'>HttpContext should implement <tt>isExternal()</tt> as returning true, all other implementers may choose based on what they represent to the system.</p>

<p style='margin-top:0;margin-bottom:10px;'>We will also provide an InternalServerContext for use by systems requiring a  an internal (<tt>isExternal()</tt> returns false) ServerContext.</p>
