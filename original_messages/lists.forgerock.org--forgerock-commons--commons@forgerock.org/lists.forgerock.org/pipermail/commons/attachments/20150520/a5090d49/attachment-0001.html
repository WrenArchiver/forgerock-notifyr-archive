                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">When I implement new functionality in CREST I have a choice: should I implement a server side Filter or Handler, or a client side Connection/ConnectionFactory? In other words, I can layer functionality by implementing a connection decorator or a handler decorator. Why does this choice exist?</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">Providing a &quot;connection&quot; abstraction on the client side may be useful for allowing client applications to logically group related requests together. For example, in a load balanced environment with loose consistency, a connection could represent a physical connection to a single server, thereby providing additional consistency guarantees since all requests will be routed to the same server (e.g. during RMW cycles). However, I'm not sure that this is useful in practice since all interactions are usually over HTTP which is connectionless.</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">The downside of exposing these client interfaces is that they cannot be extended without breaking compatibility. In particular, we cannot add convenience methods useful for scripting without breaking all existing connection implementations.</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">I think that we should consider removing these interfaces and instead introduce a concrete &quot;client&quot; class allowing us to easily extend in future. The client would be a simple wrapper around a RequestHandler:</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">

<span class="code-keyword" style="color: #000091">public</span> <span class="code-keyword" style="color: #000091">final</span> Client {
    /** The handler which will handle client requests. */
    <span class="code-keyword" style="color: #000091">private</span> <span class="code-keyword" style="color: #000091">final</span> RequestHandler handler;

    /** The context which will be forwarded to the handler with each request. */

    /**
     * Creates a <span class="code-keyword" style="color: #000091">new</span> {@code Client} which forwards requests to the provided
     * {@code RequestHandler}. This constructor allocates a {@code RootContext} and
     * passes it with each request.
     *
     * @param handler
     *            The handler which will handle client requests.
     */
    <span class="code-keyword" style="color: #000091">public</span> Client(<span class="code-keyword" style="color: #000091">final</span> RequestHandler handler) {
        <span class="code-keyword" style="color: #000091">this</span>(handler, <span class="code-keyword" style="color: #000091">new</span> RootContext());
    }

    /**
     * Creates a <span class="code-keyword" style="color: #000091">new</span> {@code Client} which forwards requests to the provided
     * {@code RequestHandler}.
     *
     * @param handler
     *            The handler which will handle client requests.
     * @param context
     *            The context which will be forwarded to the handler with each
     *            request.
     */
    <span class="code-keyword" style="color: #000091">public</span> Client(RequestHandler handler, Context context) {
        <span class="code-keyword" style="color: #000091">this</span>.handler = handler;
        <span class="code-keyword" style="color: #000091">this</span>.context = context;
    }

</span>    <span class="code-keyword" style="color: #000091">public</span> Promise&lt;Resource, ResourceException&gt; create(CreateRequest request) {
        <span class="code-keyword" style="color: #000091">return</span> handle.handleCreate(request);
    }

</span>    <span class="code-keyword" style="color: #000091">public</span> Promise&lt;Resource, ResourceException&gt; create(<span class="code-object" style="color: #910091">String</span> endpoint, <span class="code-object" style="color: #910091">String</span> newResourceId, JsonValue resource) {...}

    <span class="code-comment" style="color: #808080">// Even composite methods, like patch by query?
</span>    ...
}
