                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">did not work so well, as the generic signature of ResourceProviders and RequestHandlers expect <tt>Promise&lt;Response, ResourceException&gt;</tt> and the compiler cannot tell that InternalServerErrorException is a subclass of ResourceException. Changing the signatures of the handle methods to return <tt>Promise&lt;Response, ? extends ResourceException&gt;</tt> also does not work due to generic capture mismatches at the lower levels.</p> 
                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">To aid developers writing RequestHandlers/ResourceProviders, I propose an addition to the Response interface to include</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">

</pre> 
                                                </div>

                                            <p style="margin-top:0;margin-bottom:10px;; margin: 10px 0 0 0">that can be implemented for each ResourceException, ActionResponse, QueryResponse, and ResourceResponse. This will cut down on the static import boilerplate, but is especially important for subclasses of ResourceException which require a separate static factory method to perform the casting. Thus</p> 
                                            <div class="code panel" style="border-width: 1px;; border: 1px solid #ccc; background: #f5f5f5; font-size: 12px; line-height: 1.333; font-family: monospace; border: 1px solid #ccc; -moz-border-radius: 3px 3px 3px 3px; border-radius: 3px 3px 3px 3px; margin: 9px 0">

<span class="code-keyword" style="color: #000091">public</span> <span class="code-keyword" style="color: #000091">static</span> ResourceException newInternalServerErrorException(<span class="code-object" style="color: #910091">String</span> message, Exception cause) {
    <span class="code-keyword" style="color: #000091">return</span> <span class="code-keyword" style="color: #000091">new</span> InternalServerErrorException(message, cause);
}
</pre>

                                            </div>

<span class="code-keyword" style="color: #000091">return</span> <span class="code-keyword" style="color: #000091">new</span> InternalServerErrorException(message, cause).asPromise();
