        <p style='margin-top:0;margin-bottom:10px;'>Excerpt from the <a href="https://tools.ietf.org/html/draft-ietf-oauth-json-web-token-29#section-5.1">JWT draft spec</a>:</p>
<blockquote>
<p style='margin-top:0;margin-bottom:10px;'>   If<br/>
   present, it is RECOMMENDED that its value be "JWT" to indicate that<br/>
   this object is a JWT.  While media type names are not case-sensitive,<br/>
   it is RECOMMENDED that "JWT" always be spelled using uppercase<br/>
   characters for compatibility with legacy implementations.  Use of<br/>
   this Header Parameter is OPTIONAL.</p></blockquote>

<p style='margin-top:0;margin-bottom:10px;'>currently the JWTs created by our implementation always include the "typ" header value with lower case, for example:</p>
<div class="preformatted panel" style="border-width: 1px;"><div class="preformattedContent panelContent">
<pre>{
    "alg": "HS256", 
    "typ": "jwt", 
    "cty": "JWT"
}
</pre>
</div></div>

<p style='margin-top:0;margin-bottom:10px;'>When working with libraries such as <a href="http://blogs.msdn.com/b/webdev/archive/2014/03/28/owin-security-components-in-asp-net-openid-connect.aspx">Microsoft Owin Security OpenID Connect</a>, this will result in a failure. For interoperability purposes our JWT implementation should use the uppercase "JWT" string in the "typ" header.</p>
