+ &lt;title&gt;About ForgeRock Common REST&lt;/title&gt;
+
+ &lt;para&gt;
+  For many REST APIs that are not defined by external standards,
+  ForgeRock products provide common ways
+  to access web resources and collections of resources.
+  This section covers what is common across products.
+  Adapt the examples to your types of resources and to your deployment.
+ &lt;/para&gt;
+
+ &lt;section xml:id=&quot;about-crest-resources&quot;&gt;
+  &lt;title&gt;Common REST Resources&lt;/title&gt;
+
+  &lt;para&gt;
+   Servers generally return JSON-format resources,
+   though resource formats can depend on the implementation.
+  &lt;/para&gt;
+
+  &lt;para&gt;
+   Resources in collections can be found by their unique identifiers (IDs).
+   IDs are exposed in the resource URIs.
+   For example, if a server has a user collection under &lt;literal&gt;/users&lt;/literal&gt;,
+   then you can access a user at
+   &lt;literal&gt;/users/&lt;replaceable&gt;user-id&lt;/replaceable&gt;&lt;/literal&gt;.
+   The ID is also the value of the &quot;_id&quot; field of the resource.
+  &lt;/para&gt;
+
+  &lt;para&gt;
+   Resources are versioned using revision numbers.
+   A revision is specified in the resource's &quot;_rev&quot; field.
+   Revisions make it possible to figure out whether to apply changes
+   without resource locking and without distributed transactions.
+  &lt;/para&gt;

+
+  &lt;para&gt;
+   The &lt;emphasis&gt;action&lt;/emphasis&gt; verb is the main vehicle for extensions.
+   For example, to create a new user with HTTP POST rather than HTTP PUT,
+   you might use &lt;literal&gt;/users?_action=create&lt;/literal&gt;.
+   A server can define additional actions.
+   For example, &lt;literal&gt;/tasks/1?_action=cancel&lt;/literal&gt;.
+  &lt;/para&gt;

+   For example, &lt;literal&gt;/groups?_queryId=hasDeletedMembers&lt;/literal&gt;.
+   Stored queries can call for additional parameters.
+   The parameters are also passed in the query string.
+   Which parameters are valid depends on the stored query.
+  &lt;/para&gt;

+
+  &lt;para&gt;

+   either with an HTTP POST or with an HTTP PUT.
+  &lt;/para&gt;
+

+   To create a resource using POST, perform an HTTP POST
+   with the query string parameter &lt;literal&gt;_action=create&lt;/literal&gt;
+   and the JSON resource as a payload.
+   Accept a JSON response.
+   The server creates the identifier if not specified.
+  &lt;/para&gt;
+
+  &lt;programlisting language=&quot;http&quot;&gt;
+POST /users?_action=create  HTTP/1.1
+Host: example.com
+Accept: application/json
+Content-Length: ...
+Content-Type: application/json
+{ &lt;replaceable&gt;JSON resource&lt;/replaceable&gt; }
+  &lt;/programlisting&gt;
+
+  &lt;para&gt;
+   To create a resource using PUT, perform an HTTP PUT
+   including the case-sensitive identifier (&lt;literal&gt;_id&lt;/literal&gt;)
+   for the resource with the JSON resource as a payload.
+   Use the &lt;literal&gt;If-None-Match: *&lt;/literal&gt; header.
+   Accept a JSON response.
+  &lt;/para&gt;
+
+  &lt;programlisting language=&quot;http&quot;&gt;
+PUT /users/some-id  HTTP/1.1
+Host: example.com
+Accept: application/json
+Content-Length: ...
+Content-Type: application/json
+If-None-Match: *
+{ &lt;replaceable&gt;JSON resource&lt;/replaceable&gt; }
+  &lt;/programlisting&gt;
+
+  &lt;para&gt;
+   The &lt;literal&gt;_id&lt;/literal&gt; and content of the resource
+   depend on the server implementation.

+ &lt;section xml:id=&quot;about-crest-query&quot;&gt;
+  &lt;title&gt;Query&lt;/title&gt;
+
+  &lt;para&gt;
+   To query a resource collection
+   (or resource container if you prefer to think of it that way),
+   perform an HTTP GET and accept a JSON response, including at least
+   a &lt;literal&gt;_queryFilter&lt;/literal&gt; or &lt;literal&gt;_queryId&lt;/literal&gt; parameter.
+  &lt;/para&gt;
+
+  &lt;programlisting language=&quot;http&quot;&gt;
+GET /users?_queryFilter=true  HTTP/1.1
+Host: example.com
+Accept: application/json
+  &lt;/programlisting&gt;
+
+  &lt;para&gt;
+   The server returns the result as a JSON object
+   including a &quot;results&quot; array and other fields
+   related to the query string parameters that you specify.
+  &lt;/para&gt;
+
+  &lt;variablelist&gt;
+   &lt;title&gt;Parameters&lt;/title&gt;
+
+   &lt;para&gt;
+    You can use the following parameters.
+   &lt;/para&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_queryFilter=&lt;replaceable&gt;filter-expression&lt;/replaceable&gt;&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      Query filters request that the server return entries
+      that match the filter expression.
+      You must URL-escape the filter expression.
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      The string representation is summarized as follows.
+      Continue reading for additional explanation.
+     &lt;/para&gt;
+
+     &lt;programlisting language=&quot;none&quot;&gt;
+Expr           = OrExpr
+OrExpr         = AndExpr ( 'or' AndExpr ) *
+AndExpr        = NotExpr ( 'and' NotExpr ) *
+NotExpr        = '!' PrimaryExpr | PrimaryExpr
+PrimaryExpr    = '(' Expr ')' | ComparisonExpr | PresenceExpr | LiteralExpr
+ComparisonExpr = Pointer OpName JsonValue
+PresenceExpr   = Pointer 'pr'
+LiteralExpr    = 'true' | 'false'
+Pointer        = JSON pointer
+OpName         = 'eq' |  # equal to
+'co' |  # contains
+'sw' |  # starts with
+'lt' |  # less than
+'le' |  # less than or equal to
+'gt' |  # greater than
+'ge' |  # greater than or equal to
+STRING  # extended operator
+JsonValue      = NUMBER | BOOLEAN | '&quot;' UTF8STRING '&quot;'
+STRING         = ASCII string not containing white-space
+UTF8STRING     = UTF-8 string possibly containing white-space
+     &lt;/programlisting&gt;
+
+     &lt;para&gt;
+      A simple filter expression can represent a comparison, presence,
+      or a literal value.
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      For comparisons expression use
+      &lt;replaceable&gt;json-pointer comparator json-value&lt;/replaceable&gt;,
+      where the &lt;replaceable&gt;comparator&lt;/replaceable&gt; is one of the following.
+     &lt;/para&gt;
+
+     &lt;simplelist&gt;
+      &lt;member&gt;&lt;literal&gt;eq&lt;/literal&gt; (equals)&lt;/member&gt;
+      &lt;member&gt;&lt;literal&gt;co&lt;/literal&gt; (contains)&lt;/member&gt;
+      &lt;member&gt;&lt;literal&gt;sw&lt;/literal&gt; (starts with)&lt;/member&gt;
+      &lt;member&gt;&lt;literal&gt;lt&lt;/literal&gt; (less than)&lt;/member&gt;
+      &lt;member&gt;&lt;literal&gt;le&lt;/literal&gt; (less than or equal to)&lt;/member&gt;
+      &lt;member&gt;&lt;literal&gt;gt&lt;/literal&gt; (greater than)&lt;/member&gt;
+      &lt;member&gt;&lt;literal&gt;ge&lt;/literal&gt; (greater than or equal to)&lt;/member&gt;
+     &lt;/simplelist&gt;
+
+     &lt;para&gt;
+      For presence, use &lt;replaceable&gt;json-pointer pr&lt;/replaceable&gt;
+      to match resources where the JSON pointer is present.
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      Literal values include true (match anything) and false (match nothing).
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      Complex expressions employ &lt;literal&gt;and&lt;/literal&gt;,
+      &lt;literal&gt;or&lt;/literal&gt;,
+      and &lt;literal&gt;!&lt;/literal&gt; (not),
+      with parentheses, &lt;literal&gt;(&lt;replaceable&gt;expression&lt;/replaceable&gt;)&lt;/literal&gt;,
+      to group expressions.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_queryId=&lt;replaceable&gt;identifier&lt;/replaceable&gt;&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      Specify a query by its identifier.
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      Specific queries can take their own query string parameter arguments,
+      which depend on the implementation.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_pagedResultsCookie=&lt;replaceable&gt;string&lt;/replaceable&gt;&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The string is an opaque cookie used by the server
+      to keep track of the position in the search results.
+      The server returns the cookie in the JSON response
+      as the value of &quot;pagedResultsCookie&quot;.
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      In the request &lt;literal&gt;_pageSize&lt;/literal&gt; must also be set and non-zero.
+      You receive the cookie value from the provider on the first request,
+      and then supply the cookie value in subsequent requests
+      until the server returns a &lt;literal&gt;null&lt;/literal&gt; cookie,
+      meaning that the final page of results has been returned.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_pagedResultsOffset=&lt;replaceable&gt;integer&lt;/replaceable&gt;&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      When &lt;literal&gt;_pageSize&lt;/literal&gt; is non-zero,
+      use this as an index in the result set indicating the first page to return.
+      When the value of &lt;literal&gt;_pagedResultsOffset&lt;/literal&gt; is less than 1,
+      the server returns the next page after the last page returned.
+      When the value is greater than or equal to 1,
+      the server returns the page starting from the specified index.
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      When this value is specified and greater than or equal to zero,
+      the server calculates &quot;remainingPagedResults&quot;,
+      and provides the value as part of the response.
+      The &quot;remainingPagedResults&quot; reflects
+      an estimate of the number of remaining paged results.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_pageSize=&lt;replaceable&gt;integer&lt;/replaceable&gt;&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      Return query results in pages of this size.
+      After the initial request,
+      use &lt;literal&gt;_pagedResultsCookie&lt;/literal&gt; to page through the results.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_sortKeys=[+-]&lt;replaceable&gt;field&lt;/replaceable&gt;[,[+-]&lt;replaceable&gt;field&lt;/replaceable&gt;...]&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      Sort the resources returned based on the specified field(s),
+      either in &lt;literal&gt;+&lt;/literal&gt; (ascending, default) order,
+      or in &lt;literal&gt;-&lt;/literal&gt; (descending) order.
+     &lt;/para&gt;
+
+     &lt;para&gt;
+      The &lt;literal&gt;_sortKeys&lt;/literal&gt; parameter is not supported
+      for predefined queries (&lt;literal&gt;_queryId&lt;/literal&gt;).
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_prettyPrint=true&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      Format the body of the response.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;&lt;literal&gt;_fields=&lt;replaceable&gt;field&lt;/replaceable&gt;[,&lt;replaceable&gt;field&lt;/replaceable&gt;...]&lt;/literal&gt;&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      Return only the specified fields in the body of the response.
+     &lt;/para&gt;

+
+ &lt;section xml:id=&quot;about-crest-response-codes&quot;&gt;
+  &lt;title&gt;HTTP Status Codes&lt;/title&gt;
+
+  &lt;para&gt;
+   When working with a common REST API over HTTP,
+   you can expect the following HTTP status codes.
+   Not all servers necessarily return all status codes identified here.
+  &lt;/para&gt;
+
+  &lt;variablelist&gt;
+   &lt;varlistentry&gt;
+    &lt;term&gt;200 OK&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The request was successful and a resource returned, depending on the request.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;201 Created&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The request succeeded and the resource was created.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;400 Bad Request&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The request was malformed.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;401 Unauthorized&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The request requires user authentication.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;403 Forbidden&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      Access was forbidden during an operation on a resource.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;404 Not Found&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The specified resource could not be found, perhaps because it does not exist.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;405 Method Not Allowed&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The HTTP method is not allowed for the requested resource.
+     &lt;/para&gt;
+    &lt;/listitem&gt;
+   &lt;/varlistentry&gt;
+
+   &lt;varlistentry&gt;
+    &lt;term&gt;406 Not Acceptable&lt;/term&gt;
+    &lt;listitem&gt;
+     &lt;para&gt;
+      The request contains parameters that are not acceptable,
+      such as a resource or protocol version that is not available.
